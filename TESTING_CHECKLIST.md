## 🔧 أوامر الاختبار اليدوي

### اختبار Webhook محلياً:

```bash
# 1. تشغيل المشروع
pnpm dev

# 2. في terminal آخر، إرسال webhook محاكى
curl -X POST http://localhost:3000/api/webhooks/shopify \
  -H "Content-Type: application/json" \
  -H "x-shopify-hmac-sha256: test_hmac" \
  -d '{
    "type": "orders/paid",
    "data": {
      "object": {
        "id": "test-order-123",
        "total_price": "299.00",
        "currency": "USD",
        "note_attributes": [
          {"name": "clinic_id", "value": "YOUR_CLINIC_UUID"},
          {"name": "plan", "value": "advanced"}
        ],
        "line_items": [
          {"title": "Omar Clinic Pro - Advanced Plan"}
        ]
      }
    }
  }'
```

### اختبار الوصول:

```bash
# 1. تسجيل دخول
curl -X POST http://localhost:3000/login

# 2. محاولة الوصول إلى /dashboard/clinic
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:3000/dashboard/clinic

# 3. التحقق من الاستجابة
# - إذا كان الاشتراك نشطاً: 200 OK
# - إذا كان الاشتراك منتهياً: 307 Redirect to /subscription-expired
```

## 🚨 الأخطاء الشائعة وحلولها

### الخطأ 1: "Webhook verification failed"
**السبب:** التوقيع غير صحيح
**الحل:** تأكد من أن `SHOPIFY_WEBHOOK_SECRET` صحيح

### الخطأ 2: "clinic_id not found in order"
**السبب:** لم يتم تمرير `clinic_id` في metadata
**الحل:** تأكد من تمرير `clinic_id` من صفحة التسعير

### الخطأ 3: "Email not sent"
**السبب:** `RESEND_API_KEY` غير صحيح
**الحل:** تأكد من أن المفتاح صحيح وفعال

### الخطأ 4: "User cannot access dashboard"
**السبب:** `subscription_status` في Clerk metadata غير صحيح
**الحل:** تأكد من تحديث Clerk metadata بعد تفعيل الاشتراك
